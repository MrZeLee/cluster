# 24-meeting-bot

Self-hosted [screenappai/meeting-bot](https://github.com/screenappai/meeting-bot)
— drops a bot into Google Meet / Microsoft Teams / Zoom meetings and records
**video + audio** (WebM) to object storage. **No transcription** (this app does
not run Whisper); it is a recorder only.

## Bundles (deploy order via `dependsOn`)

1. `secrets/` — sealed `meeting-bot-s3` secret (the bot's S3 creds; must match
   the `minio-s3` creds in the standalone `25-minio` app). Generated by
   `02-secrets/install.sh` from the template under
   `02-secrets/templates/24-meeting-bot/`.
2. `02-meeting-bot/` — Helm bundle: the bot Deployment (bot container +
   Chrome-CDP sidecar), a small Redis, a ClusterIP Service on port 3000, and an
   internal-only Traefik `IngressRoute`. Pinned to `n5pro`.

The object store lives in the standalone **`25-minio`** app (namespace `minio`);
the bot uploads to it over S3 at `http://minio.minio.svc.cluster.local:9000`.

The REST API is reachable on the LAN at `https://meeting-bot.local.<zone>`
(the same `*.local.<zone>` internal pattern as uptime-kuma — not public). It is
**unauthenticated** in a self-host, so anyone who can reach that host can spawn
bots; add a Traefik basic-auth middleware if you want a lock on it.

## First deploy

1. Fill real MinIO credentials into
   `02-secrets/bundles/24-meeting-bot/meeting-bot/meeting-bot-s3.yaml`
   (copy of the template; git-ignored).
2. Run `02-secrets/install.sh` (talks to the cluster's Sealed Secrets
   controller — **you** run this, kubeseal required).
3. Commit + push. Fleet reconciles the secret → bot (MinIO comes up
   independently via `25-minio`).

## Recording a Teams meeting

```bash
curl -X POST https://meeting-bot.local.<zone>/microsoft/join \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://teams.microsoft.com/l/meetup-join/...",
    "name": "Recorder",
    "bearerToken": "unused-in-selfhost",
    "userId": "me",
    "teamId": "home",
    "botId": "00000000-0000-0000-0000-000000000000",
    "timezone": "Europe/Lisbon"
  }'
```

`GET /isbusy` shows whether the single slot is occupied; a second join while
busy returns `409 BUSY`. The bot waits in the Teams lobby until a host admits
it (`JOIN_WAIT_TIME_MINUTES`), records until the meeting ends or goes silent
(`MEETING_INACTIVITY_MINUTES`) or `MAX_RECORDING_DURATION_MINUTES`, then uploads
the `.webm` to MinIO under `meeting-bot/<userId>/<file>.webm` and deletes the
local copy.

## Recordings

Browse them in the MinIO console web UI at `https://minio.<zone>` (login with
the `meeting-bot-s3` credentials), bucket `meeting-bot`. In-cluster clients can
use the S3 API at `http://minio.minio.svc.cluster.local:9000`. See `25-minio`.
